# 01_os_and_distribution

## この章の目的

この章では、Linux環境における **OS**、**Linuxカーネル**、**ディストリビューション** の違いを確認します。

コンピューターシステムの学習では、「OS」という言葉がよく出てきます。
しかし、Linuxを使う場合は、次のような言葉を分けて理解する必要があります。

* OS
* Linux
* Linuxカーネル
* ディストリビューション
* バージョン
* アーキテクチャ

この章では、これらの言葉を単なる用語として覚えるのではなく、実際のLinux環境でコマンドを実行し、表示される情報を読み取ることで理解します。

---

## 学習の流れ

この章では、次の流れで学習します。

1. OSとは何かを確認する
2. Linuxカーネルとは何かを確認する
3. ディストリビューションとは何かを確認する
4. シェルとは何かを確認する
5. `/etc/os-release` を読んでOS情報を確認する
6. `uname` でカーネル情報を確認する
7. `/etc/os-release` と `uname` の違いを確認する
8. AlmaLinuxとUbuntuの違いを考える
9. 確認コマンドを実行して環境情報を整理する
10. 確認した情報を自分の言葉で説明する

---

## 1. OSとは何か

OSは、Operating System の略です。

OSは、コンピューターのハードウェアとアプリケーションの間に入り、コンピューター全体を管理する基本ソフトウェアです。

OSは、たとえば次のようなものを管理します。

* CPU
* メモリ
* ストレージ
* ファイル
* プロセス
* ユーザー
* ネットワーク
* 入出力装置

Windows、macOS、Linuxなどは、OSの例です。

Linuxを学ぶということは、単にコマンドを覚えることではありません。
OSがどのようにコンピューターを管理しているのかを観察することでもあります。

---

## 2. Linuxカーネルとは何か

Linuxという言葉は、厳密には **Linuxカーネル** を指します。

カーネルはOSの中心部分です。
ハードウェアに近い部分を管理し、アプリケーションやコマンドが安全にコンピューターを使えるようにします。

カーネルは、たとえば次のような役割を持ちます。

* プロセスの管理
* メモリの管理
* ファイルシステムの管理
* デバイスの管理
* ネットワーク通信の管理

普段私たちが使っている「Linux」は、カーネルだけではなく、コマンド、ライブラリ、パッケージ管理システム、設定ファイルなどを組み合わせたものです。

---

## 3. ディストリビューションとは何か

Linuxカーネルに、さまざまなソフトウェアや管理ツールを組み合わせて、使いやすい形にまとめたものを **Linuxディストリビューション** と呼びます。

代表的なディストリビューションには、次のようなものがあります。

* Ubuntu
* Debian
* AlmaLinux
* Rocky Linux
* Fedora
* Arch Linux

同じLinuxでも、ディストリビューションによって違いがあります。

たとえば、パッケージ管理コマンドが異なる場合があります。

| 系統       | 例                              | パッケージ管理      |
| -------- | ------------------------------ | ------------ |
| Debian系  | Ubuntu, Debian                 | `apt`        |
| Red Hat系 | AlmaLinux, Rocky Linux, Fedora | `dnf`, `yum` |

そのため、Linuxを使うときは「Linuxである」だけでなく、**どのディストリビューションを使っているのか**を確認することが大切です。

---
## 4. シェルとは何か

Linuxを操作するとき、私たちは多くの場合、ターミナルにコマンドを入力します。

このとき、ユーザーが入力したコマンドを受け取り、解釈し、OSに処理を依頼する役割を持つプログラムを **シェル** と呼びます。

シェルは、ユーザーとOSの間にある窓口のような存在です。

```text
ユーザー
↓
シェル
↓
OS・カーネル
↓
ハードウェア
```

たとえば、次のようなコマンドを入力したとします。

```bash
ls
```

このとき、シェルは `ls` という入力を解釈し、対応するプログラムを実行します。
その結果、現在のディレクトリにあるファイルやディレクトリの一覧が表示されます。

---

### シェルの種類

Linuxでは、いくつかの種類のシェルが使われます。

代表的なものには、次のようなものがあります。

| シェル    | 説明                     |
| ------ | ---------------------- |
| `sh`   | 基本的なUnix系シェル           |
| `bash` | 多くのLinux環境で使われる代表的なシェル |
| `zsh`  | 補完機能などが強化されたシェル        |
| `fish` | 使いやすさを重視したシェル          |

授業や教材では、主に `bash` を使うことが多いです。

現在使っているシェルは、次のコマンドで確認できます。

```bash
echo $SHELL
```

表示例：

```text
/bin/bash
```

また、現在動いているシェルのプロセスは、次のように確認できます。

```bash
ps
```

表示例：

```text
  PID TTY          TIME CMD
 1234 pts/0    00:00:00 bash
 1250 pts/0    00:00:00 ps
```

この例では、`bash` というシェルが実行中であることがわかります。

---

### シェルとコマンド入力

シェルは、単にコマンドを実行するだけではありません。

次のような機能も持っています。

* コマンドを解釈する
* 変数を扱う
* パイプ `|` を使ってコマンドをつなぐ
* リダイレクト `>` や `>>` で出力先を変える
* シェルスクリプトを実行する
* コマンド履歴を管理する

つまり、Linuxを学ぶときに使っているコマンド操作の多くは、シェルを通して行われています。

---

### OS・カーネル・シェルの関係

OS、カーネル、シェルの関係は、次のように整理できます。

| 用語          | 役割                               |
| ----------- | -------------------------------- |
| カーネル        | ハードウェアに近い部分を管理するOSの中心            |
| ディストリビューション | Linuxカーネルと各種ソフトウェアをまとめたもの        |
| シェル         | ユーザーのコマンド入力を受け取り、OSに処理を依頼するプログラム |
| ターミナル       | シェルを操作するための画面・アプリケーション           |

ここで大切なのは、**ターミナルとシェルは同じものではない**という点です。

ターミナルは、文字を入力したり表示したりするための画面です。
シェルは、その中で動いているコマンド解釈プログラムです。

```text
ターミナル
→ シェルを操作するための画面

シェル
→ コマンドを解釈して実行するプログラム
```

この違いを理解すると、Linuxの操作をより正確に説明できるようになります。

---

## 5. OS情報を確認する

Linux環境では、OSやディストリビューションの情報を `/etc/os-release` で確認できます。

```bash
cat /etc/os-release
```

表示例：

```text
NAME="AlmaLinux"
VERSION="9.5 (Teal Serval)"
ID="almalinux"
VERSION_ID="9.5"
PLATFORM_ID="platform:el9"
PRETTY_NAME="AlmaLinux 9.5 (Teal Serval)"
```

確認するポイントは次の通りです。

| 項目            | 意味             |
| ------------- | -------------- |
| `NAME`        | ディストリビューション名   |
| `VERSION`     | バージョン名         |
| `ID`          | システム内部で使われる識別名 |
| `VERSION_ID`  | バージョン番号        |
| `PRETTY_NAME` | 人間が読みやすい表示名    |

ここから、自分が使っているLinuxの種類を読み取ることができます。

---

## 6. カーネル情報を確認する

次に、カーネル情報を確認します。

```bash
uname -a
```

表示例：

```text
Linux localhost.localdomain 5.14.0-503.el9.x86_64 #1 SMP PREEMPT_DYNAMIC x86_64 GNU/Linux
```

この結果には、次のような情報が含まれています。

| 情報                      | 意味         |
| ----------------------- | ---------- |
| `Linux`                 | カーネル名      |
| `localhost.localdomain` | ホスト名       |
| `5.14.0-503.el9.x86_64` | カーネルバージョン  |
| `x86_64`                | CPUアーキテクチャ |
| `GNU/Linux`             | OSの種類を示す情報 |

より短く確認したい場合は、次のコマンドも使えます。

```bash
uname -r
```

これはカーネルバージョンだけを表示します。

```bash
uname -m
```

これはマシンのアーキテクチャを表示します。

---

## 7. `/etc/os-release` と `uname` の違い

`/etc/os-release` と `uname` は、どちらもシステム情報を確認するために使います。
しかし、見ている情報の種類が違います。

| コマンド・ファイル             | 見ているもの        |
| --------------------- | ------------- |
| `cat /etc/os-release` | ディストリビューション情報 |
| `uname -a`            | カーネル情報        |
| `uname -r`            | カーネルバージョン     |
| `uname -m`            | CPUアーキテクチャ    |

つまり、次のように整理できます。

```text
/etc/os-release
→ どのLinuxディストリビューションかを確認する

uname
→ どのカーネルで動いているかを確認する
```

この違いを理解すると、「Linux」という言葉をより正確に使えるようになります。

---

## 8. AlmaLinuxとUbuntuの違いを考える

AlmaLinuxとUbuntuは、どちらもLinuxディストリビューションです。
しかし、系統や使われる場面に違いがあります。

| 項目          | AlmaLinux   | Ubuntu           |
| ----------- | ----------- | ---------------- |
| 系統          | Red Hat系    | Debian系          |
| パッケージ管理     | `dnf`       | `apt`            |
| よく使われる場面    | サーバー、企業システム | サーバー、デスクトップ、学習環境 |
| 設定ファイルや管理方法 | Red Hat系の流儀 | Debian系の流儀       |

同じLinuxでも、環境によって使うコマンドや設定方法が変わることがあります。

そのため、教材やWeb記事を読むときは、次のことを意識する必要があります。

* その説明はどのディストリビューションを前提にしているか
* 自分の環境でも同じコマンドが使えるか
* パッケージ管理コマンドは `apt` か `dnf` か
* 設定ファイルの場所が同じか

## 9. 実行してみよう

次のコマンドを実行し、結果を確認してください。

```bash
cat /etc/os-release
uname -a
uname -r
uname -m
hostnamectl
```

`hostnamectl` でも、OSやカーネル、アーキテクチャなどの情報をまとめて確認できます。

## この章で大切なこと

Linuxを使うときは、「Linux」という一言だけで環境を判断しないことが大切です。

同じLinuxでも、ディストリビューション、バージョン、カーネル、パッケージ管理方法が異なります。

重要なのは、まず自分の環境を観察し、どのようなOS・ディストリビューション・カーネルの上で作業しているのかを説明できるようになることです。

コンピューターシステムの学習では、言葉を覚えるだけでなく、実際の環境から情報を読み取る力が重要です。

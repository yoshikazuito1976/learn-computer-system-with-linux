# 03 メモリを観察する

## この章の目的

この章では、Linux 上でメモリの状態を観察しながら、コンピュータがプログラムを動かすときにメモリを使っていることを理解します。

前の章では、CPU とプロセスについて学びました。

プロセスは、CPU によって実行されます。
しかし、プロセスは CPU だけで動いているわけではありません。

プログラムを実行するとき、そのプログラムや処理中のデータはメモリ上に置かれます。

つまり、プロセスを理解するためには、メモリの理解も必要です。

## メモリとは何か

メモリは、コンピュータが作業をするための一時的な場所です。

ストレージが「長期的に保存する場所」だとすると、メモリは「今使っているものを置く場所」です。

たとえば、アプリケーションを起動したとき、そのプログラムはストレージから読み込まれ、メモリ上に展開されます。

CPU は、メモリ上にある命令やデータを使って処理を進めます。

---

## まず観察する

Linux では、メモリの状態をコマンドで確認できます。

```bash
free -h
```

`free` コマンドは、システム全体のメモリ使用状況を表示します。

`-h` オプションをつけると、人間が読みやすい単位で表示されます。

---

## メモリ上のデータ表現

メモリには、実行中のプログラムだけでなく、処理中のデータも置かれます。

データには、文字、整数、小数、画像、音声など、さまざまな種類があります。

しかし、コンピュータの内部では、これらはすべて 0 と 1 の組み合わせとして扱われます。

たとえば、整数と小数では、メモリ上での表し方が異なります。

次のような値は、整数として表せます。

```text
100
42
-7
```

一方で、

```text
3.14
0.1
6.02 × 10^23
```

のような小数や非常に大きな数は、浮動小数点数として表されることがあります。

浮動小数点数は、2 進数版の科学的記数法と考えることができます。

10進数では、非常に大きな数を次のように書くことがあります。

```text
6.02 × 10^23
```

コンピュータの中では、これを 2 進数で考えます。

```text
1.xxxxx × 2^n
```

このように、数を次の3つに分けて表します。

| 部分 | 意味 |
| --- | --- |
| 符号部 | 正の数か負の数か |
| 指数部 | 小数点をどれだけ動かすか |
| 仮数部 | 数字の本体部分 |

たとえば、`6.02 × 10^23` のような大きな数も、2進数ではおおよそ次のような形で考えます。

```text
6.02 × 10^23 ≒ 1.xxxxx₂ × 2^78
```

ただし、メモリ上で使えるビット数には限りがあります。

そのため、浮動小数点数はいつも正確に表せるとは限りません。
近い値として保存されることがあります。

Python で確認してみましょう。

```python
x = 6.02e23

print(x)
print(type(x))
```

`6.02e23` は、Python で `6.02 × 10^23` を表す書き方です。

この章では、浮動小数点数の細かい計算方法を完全に覚える必要はありません。

大切なのは、メモリ上のデータはすべて 0 と 1 で表されており、整数と小数では表し方が違う、ということです。


## free コマンドの見方

`free -h` を実行すると、次のような項目が表示されます。

```text
              total        used        free      shared  buff/cache   available
Mem:          7.6Gi       1.2Gi       4.8Gi       100Mi       1.6Gi       6.0Gi
Swap:         2.0Gi          0B       2.0Gi
```

見るポイントは次の通りです。

| 項目 | 意味 |
| --- | --- |
| total | メモリ全体の量 |
| used | 使用中のメモリ量 |
| free | すぐに空いているメモリ量 |
| buff/cache | バッファやキャッシュとして使われているメモリ量 |
| available | 実際に利用可能と考えられるメモリ量 |
| swap | メモリが不足したときに使われる補助的な領域 |

---

## free と available の違い

`free` の値だけを見て、メモリが足りているかどうかを判断するのは少し危険です。

Linux では、空いているメモリをキャッシュとして有効活用します。

そのため、`free` が少なく見えても、実際には必要に応じて再利用できるメモリが残っている場合があります。

そこで重要になるのが `available` です。

`available` は、現在の状態で新しいプログラムが使える見込みのあるメモリ量を表します。

メモリの余裕を確認するときは、`free` だけでなく `available` に注目します。

---

## より詳しいメモリ情報を見る

Linux では、メモリ情報もテキストファイルとして確認できます。

```bash
cat /proc/meminfo
```

`/proc/meminfo` には、メモリに関する詳細な情報が表示されます。

たとえば、次のような項目があります。

```text
MemTotal
MemFree
MemAvailable
Buffers
Cached
SwapTotal
SwapFree
```

Linux では、システムの状態を `/proc` 以下の仮想ファイルから確認できます。

これは、Linux を学ぶうえで重要な考え方です。

---

## プロセスごとのメモリ使用量を見る

メモリは、実行中のプロセスによって使われます。

どのプロセスがメモリを多く使っているかを確認してみましょう。

```bash
ps aux --sort=-%mem | head
```

このコマンドは、メモリ使用率が高いプロセスを上から表示します。

`%MEM` の列を見ると、そのプロセスがメモリをどの程度使っているかを確認できます。

---

## top でリアルタイムに観察する

`top` コマンドを使うと、CPU やメモリの使用状況をリアルタイムに確認できます。

```bash
top
```

`top` の画面では、上部にシステム全体のメモリ情報が表示されます。

また、各プロセスの CPU 使用率やメモリ使用率も確認できます。

終了するときは、`q` を押します。

---

## メモリとスワップ

スワップとは、メモリが不足したときに、ストレージの一部を一時的にメモリの代わりとして使う仕組みです。

ただし、ストレージはメモリよりも遅いため、スワップを多く使う状態になると、システムの動作が重くなることがあります。

`free -h` では、スワップの使用状況も確認できます。

```bash
free -h
```

`Swap` の行に注目しましょう。

## まとめ

メモリは、コンピュータが作業をするための一時的な場所です。

プロセスは CPU によって実行されますが、実行中のプログラムやデータはメモリ上に置かれます。

Linux では、`free`、`top`、`ps`、`/proc/meminfo` などを使って、メモリの状態を観察できます。

メモリを見るときは、単に数値を暗記するのではなく、

- 何がメモリを使っているのか
- どれくらい余裕があるのか
- プロセスとメモリがどう関係しているのか

を観察することが大切です。

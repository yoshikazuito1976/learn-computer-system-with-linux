# 03 观察内存

## 本章目标

在本章中，您将观察 Linux 上的内存状态，并理解计算机运行程序时如何使用内存。

在前面的章节中，您了解了 CPU 和进程。

进程由 CPU 执行。但是，进程并不仅仅在 CPU 上运行。

当执行程序时，该程序本身和正在处理的数据都被放在内存上。

换句话说，要理解进程，您还需要理解内存。

## 什么是内存？

内存是计算机进行工作的临时场所。

如果存储是"保存长期数据的地方"，那么内存就是"放置您正在使用的内容的地方"。

例如，当您启动应用程序时，该程序从存储器加载并扩展到内存中。

CPU 使用内存上的指令和数据进行处理。

---

## 首先，让我们观察

在 Linux 上，您可以使用命令检查内存状态。

```bash
free -h
```

`free` 命令显示整个系统的内存使用情况。

使用 `-h` 选项以人类可读的单位显示。

---

## 如何阅读 free 命令

运行 `free -h` 时，显示如下项目：

```text
              total        used        free      shared  buff/cache   available
Mem:          7.6Gi       1.2Gi       4.8Gi       100Mi       1.6Gi       6.0Gi
Swap:         2.0Gi          0B       2.0Gi
```

需要注意的要点如下：

| 项目 | 意义 |
|---|---|
| total | 内存总量 |
| used | 正在使用的内存量 |
| free | 立即可用的内存量 |
| buff/cache | 用作缓冲区和缓存的内存量 |
| available | 实际可用的内存量 |
| swap | 内存不足时使用的辅助区域 |

---

## free 和 available 之间的区别

仅通过查看 `free` 值来判断是否有足够的内存是有点危险的。

在 Linux 上，空闲内存被有效地用作缓存。

因此，即使 `free` 看起来很小，实际上可能还有可以根据需要重新利用的内存。

这就是 `available` 变得重要的地方。

`available` 表示在当前状态下新程序可能使用的内存量。

检查内存可用性时，注意不仅 `free`，还要 `available`。

---

## 查看更多详细内存信息

在 Linux 上，内存信息也可以作为文本文件进行检查。

```bash
cat /proc/meminfo
```

`/proc/meminfo` 显示有关内存的详细信息。

例如，它包含如下项目：

```text
MemTotal
MemFree
MemAvailable
Buffers
Cached
SwapTotal
SwapFree
```

在 Linux 上，您可以从 `/proc` 下的虚拟文件检查系统状态。

这是学习 Linux 的重要概念。

---

## 查看每个进程的内存使用情况

内存由运行的进程使用。

让我们检查哪个进程使用最多的内存。

```bash
ps aux --sort=-%mem | head
```

此命令从顶部显示内存使用率较高的进程。

查看 `%MEM` 列，您可以看到该进程使用多少内存。

---

## 使用 top 实时观察

`top` 命令允许您实时检查 CPU 和内存使用情况。

```bash
top
```

在 `top` 屏幕上，整个系统的内存信息显示在顶部。

您还可以检查每个进程的 CPU 使用率和内存使用率。

退出时，按 `q`。

---

## 内存和交换

交换是一种机制，当内存不足时，临时使用存储的一部分作为内存的替代。

但是，由于存储速度比内存慢，当频繁使用交换时，系统性能可能会变慢。

使用 `free -h` 也可以检查交换使用情况。

```bash
free -h
```

注意 `Swap` 一行。

## 总结

内存是计算机进行工作的临时场所。

进程由 CPU 执行，但运行中的程序和数据被放在内存上。

在 Linux 上，您可以使用 `free`、`top`、`ps` 和 `/proc/meminfo` 等命令观察内存状态。

查看内存时，不要只是记忆数字，而是

- 什么在使用内存？
- 有多少空间可用？
- 进程和内存如何相关？

观察这些方面很重要。
